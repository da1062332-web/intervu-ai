# AI Coverage Rules

**Module:** Module 1.2.1 Topic Registry  
**Purpose:** Guardrails for Question Generation & Evaluation  
**Version:** 1.0.0

---

## 1. Objective

These rules define the **Coverage Contract** for any questions generated by the AI engine (Module 2). Enforcing these rules ensures that the system maintains absolute visibility over what specific subjects are being tested, allowing our assessment algorithms to compile balanced tests and provide precise, topic-specific feedback to candidates.

---

## 2. Core Rules

### Rule 1: Master Topic Reference

Every generated question **must** reference a registered topic item in the database by its unique identifier (`TopicRegistryItem.id`).

- **Why:** This links the question directly to a specific domain, topic, and subtopic, ensuring traceability.
- **Failure Case:** A question generated with a missing or unregistered topic ID will fail system validation.

### Rule 2: Concept Specificity

Every generated question **must** target and reference a specific `Concept` defined within the referenced `TopicRegistryItem.concepts` array.

- **Why:** Broadly mapping a question to a subtopic like "Arrays" is not granular enough. The question must state exactly what concept it tests (e.g., "Sliding Window") to avoid duplicate concepts in the same test.
- **Failure Case:** A question mapped to an arbitrary concept string not listed in the registry's `concepts` array for that item is rejected.

### Rule 3: No Orphan Questions

The system must reject any question that does not carry valid topic mapping metadata.

- **Why:** Orphan questions prevent the Evaluation Engine (Module 3) from scoring candidates by skill category.
- **Failure Case:** Any question without a `topicId` and a matching `concept` will fail the validation pipeline and must be regenerated.

---

## 3. Enforcement Workflow (Generation Pipeline)

During generation, these rules are validated at the **GenerationValidation** stage of the pipeline:

```
[Generated Question DTO]
          ↓
+──────────────────────────+
|  Validate Metadata Keys  | ──(Missing topicId or concept)──> [REJECT & REGENERATE]
+──────────────────────────+
          ↓
+──────────────────────────+
| Check Registry Database  | ──(topicId not in registry)─────> [REJECT & REGENERATE]
+──────────────────────────+
          ↓
+──────────────────────────+
|   Verify Concept Match   | ──(concept not in topic list)───> [REJECT & REGENERATE]
+──────────────────────────+
          ↓
[Accepted Question pool]
```
